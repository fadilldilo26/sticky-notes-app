1: How HTTP Applications Preserve State
​"The Hypertext Transfer Protocol (HTTP) is inherently stateless, meaning that each request-response cycle is independent and does not retain information from previous interactions. To preserve state and enable personalized user experiences—such as maintaining login sessions or shopping cart contents—web applications utilize mechanisms like cookies and server-side sessions. When a user authenticates, the server typically creates a unique session ID, which is stored in a cookie on the client’s browser. With every subsequent request, the browser sends this cookie back to the server, allowing the application to identify the user and retrieve their specific session data from its internal storage. This process effectively bridges the gap between disconnected HTTP requests, ensuring that the application can maintain a continuous and secure user state throughout their interaction."

​2: Procedures for Migrating to a Relational Database (e.g., MariaDB)
​"Migrating a Django application from the default SQLite database to a robust, server-based relational database like MariaDB involves a structured configuration process to ensure data integrity and system scalability. First, the database server must be installed, configured, and a new database instance created with appropriate user permissions. Next, the Django project’s settings.py file must be updated to replace the DATABASES configuration with the specific engine, name, user, password, and host information required to connect to MariaDB. Developers must also install a Python database adapter, such as mysqlclient, to enable communication between Django and the server. Once configured, running the python manage.py migrate command is essential; this triggers Django to create the necessary schema, tables, and relationships within the new MariaDB instance, successfully mapping the existing models to the production-grade database environment."
